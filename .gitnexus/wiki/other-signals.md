# Trading Signals Module Documentation

## Overview
This module contains historical trading signal data for the SOL-USDC trading pair. The signals are generated by the "imba" trading strategy and stored in JSONL (JSON Lines) format. Each signal represents a trading decision point with associated metadata.

## Signal Format
Each signal entry contains the following fields:
```json
{
    "ts": "ISO8601 timestamp",
    "signal": integer (-1 or 1),
    "position": integer (-1 or 1), 
    "source": "imba",
    "sl": null
}
```

- `ts`: Timestamp in ISO8601 format with microsecond precision
- `signal`: Direction indicator (-1 for sell/short, 1 for buy/long)
- `position`: Current position after signal (-1 for short, 1 for long)
- `source`: Strategy identifier (always "imba" in current data)
- `sl`: Stop loss level (currently unused, always null)

## File Organization
The signals are split across multiple files:
- `imba_RENKO_box0p1_lb240_sl1p5.PREV14D.jsonl`: Historical signals from previous 14 days
- `imba_RENKO_box0p1_lb240_sl1p5.jsonl`: Current signals
- `imba_fib_rebuild_lb240.jsonl`: Additional signals using Fibonacci strategy variant

## Signal Characteristics
- Signals alternate between long (1) and short (-1) positions
- Average signal frequency varies but typically multiple signals per day
- Timestamps maintain chronological order within each file
- Sub-microsecond precision used to handle multiple signals at same second

## Usage
This signal data is typically used for:
- Backtesting trading strategy performance
- Live trading signal generation
- Strategy analysis and optimization
- Historical performance analysis

## Integration Points
The signals module is typically consumed by:
- Trading execution systems
- Backtesting frameworks
- Performance analysis tools
- Strategy optimization systems

## Notes
- Stop loss functionality is provisioned in the data structure but not currently utilized
- Signal files may have overlapping time periods
- All timestamps are in UTC

## Limitations
- Signal data is specific to SOL-USDC pair only
- Historical data coverage varies by file
- No additional metadata beyond basic signal information
- Stop loss functionality not implemented
