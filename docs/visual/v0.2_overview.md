# Visual Layer v0.2 Overview

This framework builds reproducible 3D market-state-space visualizations from persisted artifacts.

## Expected Input Artifacts

- `state_space.parquet`: `ts`, `X_raw`, `Y_res`, `Z_res`
- `voxel_map.parquet`: `ts`, `voxel_id`, optional `bin_x`, `bin_y`, `bin_z`
- `voxel_stats.parquet`: per-voxel diagnostics (`pi`, `occ_eff`, persistence/instability metrics)
- `transitions_topk.parquet`: sparse voxel transitions (`from_voxel_id`, `to_voxel_id`, `p`, `rank`)
- `basins_v02_components.parquet`: basin assignment (`voxel_id`, `basin_id`) [default in v0.2]
- `basins.parquet`: legacy basin assignment fallback
- optional `basin_flows.parquet`: (`from_basin_id`, `to_basin_id`, `flow_mass`)

## Quick Start

```bash
python -m scripts.visual.build_all   --config configs/viz/v0.2.yaml   --in-dir data/runs/my_run/transitions   --out-dir data/runs/my_run/visual_v02
```

## Gate Confidence Projection

Computes state-conditioned gate-on rates and forward-projects them via Markov propagation
to answer: *"Given our current market state, what is the expected gate probability at horizon h?"*

### Concept

1. **Gate alignment**: A daily gate signal (`gate_on` = 0/1) is aligned to the 1-minute
   voxel timeline via `merge_asof` (backward direction — each minute inherits the most
   recent daily gate value).

2. **Voxel gate stats**: For each voxel, the empirical `gate_on_rate` is computed
   (fraction of time steps in that voxel where the gate was on), along with a Wilson
   confidence interval.

3. **Basin gate stats**: Gate rates are aggregated per basin, weighted by stationary
   distribution `pi`, so that high-occupancy voxels dominate the basin's rate.

4. **Markov forward projection**: Starting from a one-hot distribution on the current
   voxel (NOW), the transition operator `P` is applied `k` times (where `k` = horizon
   in minutes). The expected gate at horizon `h` is:

   `conf_h = E[gate | S_{t+h}] = sum_j  P^k(now, j) * gate_on_rate(j)`

   This is computed at both voxel level (fine-grained `P`) and basin level (coarse-grained
   basin transition matrix).

### Inputs

- All standard state-space artifacts (see above)
- Daily gate file (CSV or Parquet) configured in `gate:` section of YAML

### YAML Configuration

```yaml
gate:
  path: "data/regimes/SOLUSDT_tv5mIMBA_gate2of3_daily.csv"
  ts_col: "ts"
  col: "gate_on_2of3"
  asof_direction: "backward"

confidence:
  horizons_minutes: [5, 30, 120, 240]
  use_basins: true
  ignore_noise_basin: true
```

### CLI Usage

Standalone:
```bash
python -m scripts.visual.build_gate_confidence \
    --config configs/viz/v0.2.yaml \
    --in-dir data/runs/visual_v02_seed/transitions \
    --out-dir data/runs/visual_v02_seed/gate_confidence
```

As part of `build_all`:
```bash
python -m scripts.visual.build_all \
    --config configs/viz/v0.2.yaml \
    --in-dir data/runs/visual_v02_seed/transitions \
    --out-dir data/runs/visual_v02_seed/visual_v02_intel \
    --build-gate-confidence
```
